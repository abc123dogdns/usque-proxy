<objective>
Add CloseError-based error classification to forwardUp and forwardDown in usque-bind/bind.go, matching the usque-android reference implementation pattern.

Purpose: Currently both forwarding loops treat ALL errors as fatal, causing unnecessary reconnects on transient packet errors. By classifying errors — CloseError means the MASQUE session is dead (reconnect), other errors are transient (log and continue) — the tunnel only reconnects when truly necessary.

Output: Modified forwardUp and forwardDown functions with CloseError detection.
</objective>
